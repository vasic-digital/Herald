# Wave 4b — TOON (Token-Oriented Object Notation) Content Negotiation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` per Universal Constitution §11.4.70. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Adopt TOON (Token-Oriented Object Notation, `application/toon`) as Herald's preferred wire format with `application/json` as honest fallback. Every Wave 3a/3b live REST surface (`POST /v1/events`, `GET /v1/compliance`, `GET /v1/safety_state`) gains real content-negotiation: clients sending `Accept: application/toon` (or `Accept: */*` with the Herald-default-preference) receive a TOON-encoded body; clients sending `Accept: application/json` receive JSON. Inbound CloudEvents on `POST /v1/events` accept either content type. Tags **v0.3.0**. Closes the TOON adoption leg of the Wave 4 design.

**Architecture:** Two new files under `commons/cli/` — `contentnego.go` (Accept-header parser + codec resolver) and `toon.go` (Gin middleware + helper that consume the resolver to encode/decode the request/response body). The Wave 4a middleware chain (Brotli → Alt-Svc → flavor middleware) gains a new entry: TOON-aware response encoder that wraps `c.JSON(...)` calls in flavor handlers via a shared `cli.Respond(c, status, payload)` helper. Inbound bodies are decoded via a `cli.BindBody(c, dst)` helper that inspects `Content-Type` before deciding between `json.Unmarshal` and `toon.Unmarshal`. The flavor handlers (pherald events, cherald compliance, sherald safety) swap their three `c.JSON(...)` call sites for `cli.Respond(...)`; pherald additionally swaps `io.ReadAll → json.Unmarshal` for `cli.BindBody`. No new workspace module — TOON support lives entirely in the existing `commons` module.

**Tech Stack:** Go 1.25+; `digital.vasic.toon` (vendored at `submodules/TOON/`, NEW in W4b-T1); upstream `github.com/toon-format/toon-go` reached via the digital.vasic.toon facade (W4b-T2); `encoding/json` (stdlib, JSON fallback); `github.com/gin-gonic/gin` (existing); existing `commons/`, `commons_auth/`, Wave 4a transport substrate (`commons/cli/{h3,altsvc,brotli,serve}.go`).

**Spec reference:** [`docs/superpowers/specs/2026-05-22-http3-brotli-toon-design.md`](../specs/2026-05-22-http3-brotli-toon-design.md) (commit `c60b3fd`) — Section 2.3 (digital.vasic.toon disposition: EXTEND), Section 5 (TOON adoption design — §5.1 what is TOON, §5.2 wiring strategy, §5.3 upstream extension scope, §5.4 non-goals, §5.5 fallback ladder), Section 6.2 (e2e invariants E64..E70 — renumbered to E56..E62 in this plan because Wave 4a landed E49-E55, not E49-E63 as the design draft anticipated), Section 6.3 (mutation gates M3+M4), Section 7.4 (Wave 4b task sketch — 7 tasks), Section 8 (spec V3 r10 → r11 impact).

**Wave 4a substrate already landed:** HTTP/3 (QUIC) + Brotli + Alt-Svc + TLS 1.3 across all flavors (commit `93b496a`, tag `v0.2.0`). `commons/cli/serve.go` carries a dual TCP/UDP listener; Brotli middleware compresses responses ≥ 256 B; Alt-Svc advertises `h3=":<port>"`; TLS 1.3 mandatory; e2e_bluff_hunt at 54 invariants (E1..E55). Wave 4b is a wire-format upgrade ON TOP of that transport — zero transport regression.

**§107 anti-bluff watershed (CRITICAL):** This wave's upstream task (W4b-T2) is the highest-bluff-risk surface in Herald's history. The `digital.vasic.toon` submodule shipped a PASS-bluff revision earlier — silently delegated every `Marshal` call to `encoding/json` while serving `Content-Type: application/toon`, claiming wire-correctness while writing JSON bytes. That revision was REVERTED to an honest sentinel-error scaffold (`ErrTOONEncodingNotImplemented`) as an anti-regression marker. Wave 4b MUST replace the sentinel with REAL `toon-format/toon-go` integration. Every mutation gate + e2e invariant below was designed to catch a recurrence of the original bluff. **In particular:** E58 asserts byte-level round-trip equality (not just header presence + non-empty body); E59 asserts the TOON body is actually shorter than the equivalent JSON body for a Herald Receipt payload; M4 force-rewrites the Marshal call to fall through to JSON and asserts the e2e invariants FAIL. A green Wave 4b without those three specific assertions is itself a §11.4 PASS-bluff.

**Operator-locked decisions (recorded 2026-05-22):**

1. **TOON-default content type.** When the client sends `Accept: */*` OR `Accept` is absent, the server selects `application/toon`. When the client sends an explicit `Accept: application/json` (without TOON in the list), the server emits JSON. When the client sends an explicit `Accept: application/toon` (without JSON in the list), the server emits TOON. When BOTH are present with no quality parameters, TOON wins (Herald-default). Quality parameters (`;q=0.5`) are honored — the higher-q wins; ties go to TOON.
2. **Inbound CloudEvents accept both content types.** `POST /v1/events` reads `Content-Type` and decodes TOON when it is `application/toon`, JSON when it is `application/json` (and JSON as default when Content-Type is absent, for backwards-compat with curl `-d '...'` usage). A malformed body returns 400 with the codec name in the error detail (`toon: unmarshal failed: ...` or `json: unmarshal failed: ...`).
3. **Server-default opt-out.** Operators who want JSON-default behaviour (e.g., transition periods, legacy client compatibility) set `HERALD_CODEC_DEFAULT=json`. The default is `toon`. The flag is read once at serve startup; not per-request.
4. **Encoder-failure fallback.** If `toon.Marshal(payload)` returns a non-nil error (because the payload uses a Go type the codec can't represent — slice-of-interface, channel, function, etc.), the server falls back to JSON encoding AND emits `X-Herald-Codec-Fallback: encoder_error` on the response header AND increments a `<flavor>_codec_fallback_total{reason="encoder_error"}` Prometheus counter. The fallback is observable, never silent.
5. **No TOON for `/metrics` or `/healthz`/`/readyz`.** Prometheus `/metrics` MUST stay in its native text format. The healthz/readyz routes return tiny constant JSON and are NOT bandwidth-sensitive — keep them simple JSON regardless of Accept header. Wave 4b's negotiation applies only to the three `/v1/*` business routes.
6. **No TOON request bodies for inbound webhooks from third parties.** Telegram/Slack/etc. payloads stay JSON because we don't control the sender. The inbound TOON path is only for Herald-CLI-style callers (operators, agents, CI) who explicitly opt in.

```go
func TestMarshalUnmarshal_RoundTrip(t *testing.T) {
    type row struct {
        ID   int    `json:"id"   toon:"id"`
        Name string `json:"name" toon:"name"`
    }
    original := []row{{1, "alice"}, {2, "bob"}}

    encoded, err := toon.Marshal(original)
    if err != nil {
        t.Fatalf("Marshal: %v", err)
    }
    if len(encoded) == 0 {
        t.Fatal("Marshal returned empty bytes (bluff regression: previous revision returned the equivalent of nil)")
    }

    var decoded []row
    if err := toon.Unmarshal(encoded, &decoded); err != nil {
        t.Fatalf("Unmarshal: %v", err)
    }
    if !reflect.DeepEqual(original, decoded) {
        t.Fatalf("round-trip mismatch:\n  original: %+v\n  decoded:  %+v", original, decoded)
    }
}

func TestMarshal_NotJSONBluff(t *testing.T) {
    // §107 anti-bluff: explicitly assert the encoded bytes are NOT JSON.
    // A regression of the original PASS-bluff would produce JSON bytes
    // here. The TOON spec uses a tabular header `users[N]{cols}` line
    // that is unambiguously NOT JSON.
    encoded, err := toon.Marshal(map[string]any{"x": 1})
    if err != nil {
        t.Fatal(err)
    }
    if bytes.HasPrefix(encoded, []byte("{")) || bytes.HasPrefix(encoded, []byte("[")) {
        t.Fatalf("encoded body looks like JSON (bluff regression): %q", encoded)
    }
}
```

The second test is the watershed assertion — a re-introduction of the JSON-fallback bluff would produce bytes starting with `{` or `[`, and this test would catch it. Per §107, this assertion stays in the upstream test suite permanently.

```go
// Wave 4b — Accept-header content negotiation primitive.
//
// Pure functions over header strings + a typed Codec enum. No Gin coupling
// here; the Gin glue lives in commons/cli/toon.go which calls into this
// package's NegotiateResponseCodec / NegotiateRequestCodec helpers.
//
// Policy (per W4b operator decision 1):
//   - Accept: */*           ⇒ server default (Herald default = TOON; operator can flip via HERALD_CODEC_DEFAULT)
//   - Accept: application/toon (no q)  ⇒ TOON
//   - Accept: application/json (no q)  ⇒ JSON
//   - Accept: both, no q     ⇒ TOON wins (Herald-default tie-break)
//   - Accept: both, with q   ⇒ higher q wins
//   - Accept: unsupported    ⇒ JSON safe fallback (never serve a content-type we can't speak)
//   - Accept: missing/malformed ⇒ server default
//
// §107 anti-bluff: the "safe fallback to JSON" rung is observable — a
// caller asking for `application/xml` gets `Content-Type: application/json`
// in the response (NOT `application/xml` with JSON bytes inside, which
// would be a §11.4 PASS-bluff). The honest fallback always matches the
// emitted bytes.
package cli

import (
    "strconv"
    "strings"
)

// Codec is the typed enum of wire formats Herald can encode/decode.
// The zero value (CodecUnknown) deliberately exists to force callers to
// handle the "we couldn't pick anything" case rather than defaulting
// silently to JSON inside the negotiator (the caller is in a better
// position to choose, e.g., flavor-specific defaults).
type Codec int

const (
    CodecUnknown Codec = iota
    CodecJSON
    CodecTOON
)

// MediaType returns the canonical Content-Type string for a Codec.
// Used by both response encoding (set Content-Type header) and request
// decoding logging / observability.
func MediaType(c Codec) string {
    switch c {
    case CodecTOON:
        return "application/toon"
    case CodecJSON:
        return "application/json"
    }
    return "application/octet-stream"
}

// NegotiateResponseCodec inspects the client's Accept header + the
// server-default codec name ("toon" or "json", from HERALD_CODEC_DEFAULT
// or hardcoded "toon" if unset) and returns the codec the response will
// use.
//
// The Accept header parsing follows RFC 7231 §5.3.2 (media-range with
// optional q parameter). We do NOT implement subtype wildcards beyond
// `*/*` — Herald only ever serves `application/{toon,json}` so a client
// asking for `application/*` would still need to be steered to TOON or
// JSON; we treat `application/*` as `*/*` (server default).
func NegotiateResponseCodec(acceptHeader, serverDefault string) Codec {
    if strings.TrimSpace(acceptHeader) == "" {
        return defaultFromString(serverDefault)
    }
    parts := strings.Split(acceptHeader, ",")
    bestTOON, bestJSON := -1.0, -1.0
    sawWildcard := false
    sawSupported := false
    for _, p := range parts {
        mediaType, q := parseAcceptPart(p)
        switch mediaType {
        case "application/toon":
            sawSupported = true
            if q > bestTOON {
                bestTOON = q
            }
        case "application/json":
            sawSupported = true
            if q > bestJSON {
                bestJSON = q
            }
        case "*/*", "application/*":
            sawWildcard = true
        }
    }
    switch {
    case bestTOON > bestJSON:
        return CodecTOON
    case bestJSON > bestTOON:
        return CodecJSON
    case bestTOON >= 0 && bestJSON >= 0:
        // Tie — Herald-default tie-break.
        return defaultFromString(serverDefault)
    case bestTOON >= 0:
        return CodecTOON
    case bestJSON >= 0:
        return CodecJSON
    case sawWildcard:
        return defaultFromString(serverDefault)
    case !sawSupported:
        // Client asked only for things we can't speak (e.g., application/xml).
        // Honest fallback: we ARE going to send JSON bytes; advertise so.
        return CodecJSON
    }
    return defaultFromString(serverDefault)
}

// NegotiateRequestCodec inspects the client's Content-Type header on an
// inbound request body and returns the codec to decode with. Default
// when Content-Type is absent is CodecJSON (backwards-compat with the
// curl `-d '...'` usage pattern that omits Content-Type).
func NegotiateRequestCodec(contentType string) Codec {
    // Strip parameters (e.g., "; charset=utf-8") — we only care about the
    // base media type.
    mt := strings.TrimSpace(strings.SplitN(contentType, ";", 2)[0])
    switch strings.ToLower(mt) {
    case "application/toon":
        return CodecTOON
    case "application/json", "":
        return CodecJSON
    }
    // Unknown Content-Type: refuse to guess. Caller decides whether to
    // error or to fall back to JSON. Return CodecUnknown explicitly.
    return CodecUnknown
}

// parseAcceptPart returns the media type + q parameter (defaulting to
// 1.0 if absent or malformed) for one Accept-header entry like
// "application/toon;q=0.5".
func parseAcceptPart(part string) (string, float64) {
    segs := strings.Split(part, ";")
    mediaType := strings.ToLower(strings.TrimSpace(segs[0]))
    q := 1.0
    for _, s := range segs[1:] {
        kv := strings.SplitN(strings.TrimSpace(s), "=", 2)
        if len(kv) == 2 && strings.ToLower(kv[0]) == "q" {
            if v, err := strconv.ParseFloat(strings.TrimSpace(kv[1]), 64); err == nil {
                q = v
            }
        }
    }
    return mediaType, q
}

// defaultFromString maps the HERALD_CODEC_DEFAULT env-string to a Codec.
// Unknown strings fall back to CodecTOON (the Herald default per the
// operator mandate).
func defaultFromString(s string) Codec {
    switch strings.ToLower(strings.TrimSpace(s)) {
    case "json":
        return CodecJSON
    case "toon", "":
        return CodecTOON
    }
    return CodecTOON
}
```

## Task 4: `commons/cli/toon.go` — Gin TOON encode/decode middleware

**Files:**
- Create: `commons/cli/toon.go`, `commons/cli/toon_test.go`
- Modify: `commons/cli/serve.go` (read `HERALD_CODEC_DEFAULT` env var at startup; pass `ServerCodecDefault` into the middleware chain — minimal change, NO transport regression)

This task wires the negotiation primitive into Gin. Two consumer-facing helpers:

- `cli.Respond(c *gin.Context, status int, payload any)` — replaces `c.JSON(status, payload)` in flavor handlers. Inspects `Accept`, picks codec, encodes payload, writes `Content-Type` + body. On encoder failure, falls back to JSON with `X-Herald-Codec-Fallback: encoder_error` header + counter increment.
- `cli.BindBody(c *gin.Context, dst any) error` — replaces `c.ShouldBindJSON(dst)` (or `io.ReadAll` + `json.Unmarshal`) in flavor handlers. Inspects `Content-Type`, picks codec, decodes into `dst`. On decode error, returns a wrapped error tagged with the codec name (`toon: unmarshal: ...` / `json: unmarshal: ...`) — caller maps to 400.

The middleware chain in `serve.go` does NOT install these as Gin handlers (they're per-handler helpers, not chain-style middleware). The `serve.go` change is minimal: read `HERALD_CODEC_DEFAULT` once at startup and stash it on a package-level variable that `Respond` reads. Documenting that explicitly because it's tempting to over-engineer this as a middleware factory.

```go
// Wave 4b — TOON-aware Gin response/request helpers.
//
// Replaces flavor-handler c.JSON(...) + c.ShouldBindJSON(...) calls with
// codec-aware wrappers driven by commons/cli/contentnego.go. Default
// codec is TOON (per the operator mandate); JSON is the honest fallback
// path advertised in the response's Content-Type header — never a
// silent swap.
//
// §107 anti-bluff watershed: the original digital.vasic.toon revision
// silently delegated to encoding/json while serving Content-Type:
// application/toon. That bluff is recurrence-protected here by:
//
//   1. Respond reads the chosen codec ONCE and uses ONLY that codec's
//      Marshal — no fallback unless the codec returns an error.
//   2. On encoder failure, the response carries
//      X-Herald-Codec-Fallback: encoder_error AND a Prometheus counter
//      increment. The fallback is observable, not silent.
//   3. The matching e2e invariants (E58/E59) assert round-trip equality
//      AND wire-size comparison against JSON — a buggy encoder that
//      emits JSON-bytes-with-TOON-header would FAIL both.
package cli

import (
    "bytes"
    "encoding/json"
    "fmt"
    "io"
    "net/http"
    "os"
    "strings"
    "sync"

    "github.com/gin-gonic/gin"
    "github.com/prometheus/client_golang/prometheus"

    "digital.vasic.toon"
)

// serverCodecDefault is the server's preferred codec name read once from
// HERALD_CODEC_DEFAULT at serve startup. Default "toon" per Herald
// mandate. Tests reset it via the bare assignment.
var (
    serverCodecDefault    = readServerCodecDefault()
    codecFallbackCounter  *prometheus.CounterVec
    codecFallbackInitOnce sync.Once
)

func readServerCodecDefault() string {
    s := strings.ToLower(strings.TrimSpace(os.Getenv("HERALD_CODEC_DEFAULT")))
    if s == "json" {
        return "json"
    }
    return "toon"
}

// initCodecMetrics is called lazily on the first Respond invocation that
// emits a fallback header. Pre-registering would require knowing which
// flavor is running at package-init time; lazy init keeps the
// dependency one-way (commons/cli → prometheus, not vice versa).
func initCodecMetrics() {
    codecFallbackInitOnce.Do(func() {
        codecFallbackCounter = prometheus.NewCounterVec(prometheus.CounterOpts{
            Name: "herald_codec_fallback_total",
            Help: "Number of times the response encoder fell back to JSON because the preferred TOON codec returned an error. reason='encoder_error' is the only label value emitted today.",
        }, []string{"reason"})
        _ = prometheus.Register(codecFallbackCounter) // ignore double-register error in tests
    })
}

// Respond writes payload to c with status, picking TOON or JSON per the
// client's Accept header + the server-default codec. Replaces
// c.JSON(status, payload) in flavor handlers.
//
// Encoder-failure semantics: if the chosen codec returns a non-nil error
// (e.g., TOON can't represent the payload's Go type), Respond falls back
// to JSON, sets X-Herald-Codec-Fallback: encoder_error, and increments
// the Prometheus counter. This is observable, not silent — the
// alternative (returning 500 InternalServerError) would break clients
// for what is fundamentally a server bug, while a silent fallback would
// hide the bug entirely.
func Respond(c *gin.Context, status int, payload any) {
    codec := NegotiateResponseCodec(c.GetHeader("Accept"), serverCodecDefault)
    if codec == CodecTOON {
        b, err := toon.Marshal(payload)
        if err == nil && len(b) > 0 {
            c.Data(status, MediaType(CodecTOON), b)
            return
        }
        // Encoder error → fall back to JSON with observability.
        initCodecMetrics()
        if codecFallbackCounter != nil {
            codecFallbackCounter.WithLabelValues("encoder_error").Inc()
        }
        c.Header("X-Herald-Codec-Fallback", "encoder_error")
        // Fall through to JSON below.
    }
    // JSON path (explicit Accept or fallback from TOON).
    b, err := json.Marshal(payload)
    if err != nil {
        // Both codecs failed — 500 with a typed error so the operator can debug.
        c.JSON(http.StatusInternalServerError, gin.H{
            "error":  "encoder failure (both TOON and JSON failed)",
            "detail": err.Error(),
        })
        return
    }
    c.Data(status, MediaType(CodecJSON), b)
}

// BindBody reads c.Request.Body and decodes it into dst per Content-Type.
// TOON for application/toon; JSON for application/json or absent
// Content-Type. Unknown Content-Type returns an error tagged with the
// unknown type so the caller can map to 415 Unsupported Media Type.
//
// Errors:
//   - empty body returns nil + dst unchanged (caller decides if that's OK)
//   - codec.Unmarshal failure returns "toon: unmarshal: ..." or "json: unmarshal: ..."
//   - unknown content-type returns "unsupported content-type: <type>"
func BindBody(c *gin.Context, dst any) error {
    body, err := io.ReadAll(c.Request.Body)
    if err != nil {
        return fmt.Errorf("read body: %w", err)
    }
    if len(body) == 0 {
        return nil
    }
    codec := NegotiateRequestCodec(c.GetHeader("Content-Type"))
    switch codec {
    case CodecTOON:
        if err := toon.Unmarshal(body, dst); err != nil {
            return fmt.Errorf("toon: unmarshal: %w", err)
        }
    case CodecJSON:
        if err := json.Unmarshal(body, dst); err != nil {
            return fmt.Errorf("json: unmarshal: %w", err)
        }
    case CodecUnknown:
        return fmt.Errorf("unsupported content-type: %q", c.GetHeader("Content-Type"))
    }
    return nil
}

// BindRaw is the variant for handlers that want the raw bytes for
// downstream type-aware parsing (e.g., pherald's runner takes raw
// CloudEvent bytes). Returns the bytes + the negotiated codec for
// handler-side dispatch.
func BindRaw(c *gin.Context) ([]byte, Codec, error) {
    body, err := io.ReadAll(c.Request.Body)
    if err != nil {
        return nil, CodecUnknown, fmt.Errorf("read body: %w", err)
    }
    codec := NegotiateRequestCodec(c.GetHeader("Content-Type"))
    return body, codec, nil
}
```

```go
func EventsHandler(r *runner.Runner) gin.HandlerFunc {
    return func(c *gin.Context) {
        claimsAny, ok := c.Get(commons_auth.ContextKeyClaims)
        if !ok {
            cli.Respond(c, http.StatusUnauthorized, gin.H{"error": "no auth claims in context"})
            return
        }
        claims, claimsOK := claimsAny.(map[string]any)
        if !claimsOK {
            cli.Respond(c, http.StatusUnauthorized, gin.H{"error": "auth claims wrong shape"})
            return
        }
        raw, codec, err := cli.BindRaw(c)
        if err != nil {
            cli.Respond(c, http.StatusBadRequest, gin.H{"error": "read body", "detail": err.Error()})
            return
        }
        // The Runner's EventParser expects JSON-shaped CloudEvent bytes.
        // If the wire was TOON, transcode HERE — the Runner stays codec-
        // agnostic; codec adaptation is a transport-layer concern.
        if codec == cli.CodecTOON {
            transcoded, terr := transcodeTOONToJSON(raw)
            if terr != nil {
                cli.Respond(c, http.StatusBadRequest, gin.H{"error": "toon: transcode to json failed", "detail": terr.Error()})
                return
            }
            raw = transcoded
        }
        rcpt, err := r.Run(c.Request.Context(), raw, claims)
        if err != nil {
            cli.Respond(c, mapErrorToStatus(err), gin.H{"error": err.Error()})
            return
        }
        if rcpt == nil {
            cli.Respond(c, http.StatusInternalServerError, gin.H{"error": "runner returned nil receipt"})
            return
        }
        if rcpt.WasReplay {
            c.Header("X-Herald-Replay", "true")
            cli.Respond(c, http.StatusOK, rcpt)
            return
        }
        cli.Respond(c, http.StatusAccepted, rcpt)
    }
}

// transcodeTOONToJSON decodes raw TOON bytes into a generic map then
// re-encodes as JSON for the Runner's EventParser. This is a temporary
// adaptation: in a future wave the EventParser will gain native TOON
// support and this transcoding becomes unnecessary. For Wave 4b the
// transcoding pattern keeps the Runner contract narrow.
func transcodeTOONToJSON(raw []byte) ([]byte, error) {
    var v any
    if err := toon.Unmarshal(raw, &v); err != nil {
        return nil, err
    }
    return json.Marshal(v)
}
```
